Pizza Delivery App

Introduction

This project intends to bring a redesign concept of the existing Canadian 241 Pizza Singapore mobile application. The main functionalities of the application allow users sign in/sign up, browse, order and purchase food items from the restaurant. With its confusing menu, unaesthetic look and many others, the redesign of the application aims to improve usability goals and user experience goals.

Process

Competitor Analysis: Pizza Hut Singapore, Sarpino’s Pizza Singapore, ThePizzaConpany 1112

Data Gathering: user feedback analysis, online survey, think-aloud observation

Establishing Requirements: personas, scenarios, use cases

Prototyping: low-fi prototype, hi-fi prototype

Evaluation: Heuristic evaluation, cognitive walkthrough

Conceptual Model

The key functionality of the application is food ordering, which is similar to how a pizza restaurant operates. The application database will retrieve and display the available food items on the menu, users can make selections to place order and checkout, then the application will send requests to the kitchen to prepare the food, and in the end users will get the food delivered or collect by themselves. Therefore, the metaphor of the application is a pizza restaurant, whereby users can have a similar restaurant experience on the mobile application.

The Design

Registration

To improve sign up efficiency and member conversion rate, 9 compulsory fields on the registration form are reduced down to 3.

Sign In

Users will access the application via OTP, memorising the password is no longer a headache.

Menu

Inappropriate iconography mapping and unorganised menu items are converted to intuitive menu tabs on home screen.

Tracker

Keep users updated with the order status, get assistance anytime with the built-in contact support button.

Gamification

More fun and engaging with the point reward system.
